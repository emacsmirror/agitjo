* Changelog

** Unreleased

** [[https://codeberg.org/halvin/agitjo/src/tag/v1.0.0][1.0.0]] - 2025-10-26

This is the first stable release of Agitjo.

*** Changed

- Encode descriptions in base64, now that Codeberg has adopted Forgejo
  version 13 with support for decoding base64.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.5.1][0.5.1]] - 2025-10-03

*** Fixed

- Fix a case where a failure to post (due to network errors or otherwise) would
  unexpectedly cause the pushed description to not change from that point
  onward.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.5.0][0.5.0]] - 2025-09-29

*** Added

- Add suffix command ~agitjo-visit-last-pushed-pullreq~ to ~agitjo-push~ for
  visiting the last-pushed pull request.
- Add missing =Package-Requires= dependency on the ~markdown-mode~ package.

*** Changed

- Change behavior of ~agitjo--topic-variable-infix~ ("Session/topic" option)
  such that invoking it will unset the variable instead of prompting again when
  it is already set, which makes it consistent with other ~transient-option~
  infixes.
- Revise descriptions in menu to be more consistent, and also to provide some
  subtle visual cues via fontifications.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.4.0][0.4.0]] - 2025-09-02

*** Added

- Add option to mark new pull request as a draft.
- Insert commit message body of the most recent source commit in new pull
  request descriptions.

*** Changed

- Split up options section into two groups: "Session options" and "New pull
  request options".
- Don't include front matter metadata from Markdown templates in new pull
  request descriptions.
- Use Markdown template from "main" remote branch instead of the current branch.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.3.1][0.3.1]] - 2025-08-30

*** Fixed

- Fix an error that occurs when no pull request template exists in the
  repository.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.3.0][0.3.0]] - 2025-08-29

This version removes the experimental warning label.  Stability is not quite yet
guaranteed, but radical changes to the code should be less likely now.

*** Added

- Add suffix command ~agitjo-push-pullreq-local-branch~ to menu for pushing pull
  requests from local branch.
- Add suffix command ~agitjo-push-pullreq-local-branch-or-ref~ to menu for
  pushing pull requests from local branch or reference.
- Add facilities for saving, recovering, and discarding pull request description
  drafts.
- Insert markdown templates for new PR drafts when available, like
  =.forgejo/PULL_REQUEST_TEMPLATE.md=.

*** Changed

- Make the current topic persist per repository instead of globally.
- Make the ~git push~ operation synchronous when submitting a new PR, to prevent
  data loss from losing the draft post buffer in case push fails.
- No longer set default values for the ~source~ and ~target~ slot for
  suffix command ~agitjo-push-pullreq~.
- Don't make ~agitjo-push~ inapt if there's no current branch.
- Remove function ~agitjo-pullreq-refspec~; it is now an internal method.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.2.0][0.2.0]] - 2025-08-28

*** Added

- Add transient suffix class ~agitjo-push-pullreq-suffix~.
- Support setting upstream on invocation of
  ~agitjo-push-pullreq-current-to-upstream~ if an associated upstream is not
  found.
- Define and add suffix command ~agitjo-push-pullreq~ to menu, for pushing pull
  requests from some local branch to some remote branch.
- Define and add suffix command ~agitjo-push-pullreq-current~ to menu, for
  pushing pull requests from current branch to some remote branch.

*** Changed

- Make the topic option optional, no longer prompting for a value when opening
  the menu.
- Make the topic option persistent per Emacs session.
- Rename various existing facilities.

** [[https://codeberg.org/halvin/agitjo/src/tag/v0.1.0][0.1.0]] - 2025-08-26

Initial release.
